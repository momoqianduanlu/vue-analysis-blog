# new Vue 发生了什么(initState)

上一章，我们已经从入口代码做了分析，分析了new Vue过程中做的初始化工作，虽然当前小节标题是new Vue发生了什么，但是当前小节并不会再重复去分析new Vue这个过程了，可以认为我们已经掌握了new Vue的初始化过程了。

在做当前小节的分析之前，我们来定一下本章的学习目标，也就是数据驱动，通俗的来讲我们这一章的目标就是要分析清楚 模板和数据是如何渲染成真实的dom的，这一句话中有三个关键词，模板，数据，真实dom，现在，让我们先从数据开始做分析，我们定义了数据我们该如何拿到数据。

我们建立了一个简单的工程，来研究vue源码，

import Vue from 'vue'

var app = new Vue({
  el: '#app',
  data: {
    message: 'Hello Vue!'
  },
  mounted () {
    console.log(this.message)
  }
})

那么我们在这里 import Vue from 'vue'其实是在 node_modules中引入的，我们在 node_modules找到 vue 文件夹，在这个文件夹中有 package.json文件，

"main": "dist/vue.runtime.common.js",
"module": "dist/vue.runtime.esm.js",

这样来看我们要引入的文件应该在 vue/dist中，但我们如果是 vue-cli初始化的工程，千万不要忘了 webpack的配置，他可能在 alias别名配置中引入了其他版本的 vue ，在 vue-demo 中我们的 cli 版本是 vue-cli3，因为我们要分析 Runtime + Complier版本的 vue，所以我们要在 vue.config.js中进行配置

module.exports = {
  runtimeCompiler: true
}

现在我们打开我们 vue-demo 中的 vue.esm.js，这个js文件就是接下来我们要仔细分析的vue源码文件了，在vue2.5.17-beta版本的 vue 源码中我们分析到了 vue 的 _init() 方法，现在我们在 vue-demo 的 vue-esm.js 中去寻找 _init() 方法，搜到 _init() 方法以后，我们打一个 debugger，

function initMixin (Vue) {
  Vue.prototype._init = function (options) {
    // _init debugger
    debugger
    var vm = this;
    // a uid
    vm._uid = uid$3++;

    var startTag, endTag;
    /* istanbul ignore if */
    if (process.env.NODE_ENV !== 'production' && config.performance && mark) {
      startTag = "vue-perf-start:" + (vm._uid);
      endTag = "vue-perf-end:" + (vm._uid);
      mark(startTag);
    }

    // a flag to avoid this being observed
    vm._isVue = true;
    // merge options
    if (options && options._isComponent) {
      // optimize internal component instantiation
      // since dynamic options merging is pretty slow, and none of the
      // internal component options needs special treatment.
      initInternalComponent(vm, options);
    } else {
      vm.$options = mergeOptions(
        resolveConstructorOptions(vm.constructor),
        options || {},
        vm
      );
    }
    /* istanbul ignore else */
    if (process.env.NODE_ENV !== 'production') {
      initProxy(vm);
    } else {
      vm._renderProxy = vm;
    }
    // expose real self
    vm._self = vm;
    initLifecycle(vm);
    initEvents(vm);
    initRender(vm);
    callHook(vm, 'beforeCreate');
    initInjections(vm); // resolve injections before data/props
    initState(vm);
    initProvide(vm); // resolve provide after data/props
    callHook(vm, 'created');

    /* istanbul ignore if */
    if (process.env.NODE_ENV !== 'production' && config.performance && mark) {
      vm._name = formatComponentName(vm, false);
      mark(endTag);
      measure(("vue " + (vm._name) + " init"), startTag, endTag);
    }

    if (vm.$options.el) {
      vm.$mount(vm.$options.el);
    }
  };
}

接下来我们开启 vue-demo 的服务，我们去做一下调试，一步一步进行调试以后，我们会发现当我们执行到 if (vm.$options.el) {} 的时候我们的页面渲染还没有发生变化，而当我们执行完vm.$mount(vm.$options.el) 的时候，我们的插值已经渲染成了我们的数据，

# initState(vm)分析

通常我们在写 vue 的时候，我们定义了 data 后我们还会写生命周期函数，比如 mounted，在mounted里面我们去 console.log(this.message)，可以看到我们打印出了 this.message，

那为什么我们可以访问到这个 message？

是不是想当然的认为我们在data 中定义了message，那么在mounted 中就可以访问到 message？

实际上 vue 在 mounted 函数中是做了一些事情的，下面我们来具体分析一下他是怎么做到这些事情的，

在源码中，可以看到初始化Vue.prototype._init的时候有一个 initState(vm)，

import { initState } from './state'

打开它，找到initState，

export function initState (vm: Component) {
  vm._watchers = []
  const opts = vm.$options
  if (opts.props) initProps(vm, opts.props)
  if (opts.methods) initMethods(vm, opts.methods)
  if (opts.data) {
    initData(vm)
  } else {
    observe(vm._data = {}, true /* asRootData */)
  }
  if (opts.computed) initComputed(vm, opts.computed)
  if (opts.watch && opts.watch !== nativeWatch) {
    initWatch(vm, opts.watch)
  }
}

这个 state 做了那些事情呢，首先他拿到了 $options，如果有 props就初始化props，如果有 methods 就初始化 methods，如果有data 就初始化 data，那我们来分析 initData(vm)，

function initData (vm: Component) {
  let data = vm.$options.data
  data = vm._data = typeof data === 'function'
    ? getData(data, vm)
    : data || {}
  if (!isPlainObject(data)) {
    data = {}
    process.env.NODE_ENV !== 'production' && warn(
      'data functions should return an object:\n' +
      'https://vuejs.org/v2/guide/components.html#data-Must-Be-a-Function',
      vm
    )
  }
  // proxy data on instance
  const keys = Object.keys(data)
  const props = vm.$options.props
  const methods = vm.$options.methods
  let i = keys.length
  while (i--) {
    const key = keys[i]
    if (process.env.NODE_ENV !== 'production') {
      if (methods && hasOwn(methods, key)) {
        warn(
          `Method "${key}" has already been defined as a data property.`,
          vm
        )
      }
    }
    if (props && hasOwn(props, key)) {
      process.env.NODE_ENV !== 'production' && warn(
        `The data property "${key}" is already declared as a prop. ` +
        `Use prop default value instead.`,
        vm
      )
    } else if (!isReserved(key)) {
      proxy(vm, `_data`, key)
    }
  }
  // observe data
  observe(data, true /* asRootData */)
}

initData 他其实很简单，let data = vm.$options.data 他从 $options.data 中拿到 data，拿到以后他会做一个判断，如果 data 是一个函数的话我们就调用 getData【因为通常的话我们建议是写一个function而不是一个对象】，如果不是那么 data 就是 data || {}，

export function getData (data: Function, vm: Component): any {
  // #7573 disable dep collection when invoking data getters
  pushTarget()
  try {
    return data.call(vm, vm)
  } catch (e) {
    handleError(e, vm, `data()`)
    return {}
  } finally {
    popTarget()
  }
}

getData他去 call 了一下这个 data，调用完 getData 函数以后返回一个data，他是一个对象，这个 data不仅赋值给临时变量 data 还赋值给了 vm._data，

继续往下的话就是判断 data 是不是一个对象，判断在开发环境下我们必须返回一个对象，如果不是的话就报一个警告，

接下来，他会拿到 data 的 keys，props 和 methods，然后去做循环对比，什么意思？

const keys = Object.keys(data)
const props = vm.$options.props
const methods = vm.$options.methods

let i = keys.length
while (i--) {
    const key = keys[i]
    if (process.env.NODE_ENV !== 'production') {
        if (methods && hasOwn(methods, key)) {
            warn(
                `Method "${key}" has already been defined as a data property.`,
                vm
            )
        }
    }
    if (props && hasOwn(props, key)) {
        process.env.NODE_ENV !== 'production' && warn(
            `The data property "${key}" is already declared as a prop. ` +
            `Use prop default value instead.`,
            vm
        )
    } else if (!isReserved(key)) {
        proxy(vm, `_data`, key)
    }
}

也就是当我们在data 中定义了message 后，我们就不能在 props 和 methods 中定义 message了，为什么不能用，为什么他们会冲突，是因为他们最终会挂载到 vm上，也就是当前的实例上，这个 message最终会挂到 this 也就是 vm 上，所以我们可以通过 this.message访问也可以通过 this._data.message进行访问，

那他到底是怎么实现的呢？

proxy(vm, `_data`, key)

实际上最终就通过 proxy，proxy 顾明思义他是一层代理，

const sharedPropertyDefinition = {
  enumerable: true,
  configurable: true,
  get: noop,
  set: noop
}

export function proxy (target: Object, sourceKey: string, key: string) {
  sharedPropertyDefinition.get = function proxyGetter () {
    return this[sourceKey][key]
  }
  sharedPropertyDefinition.set = function proxySetter (val) {
    this[sourceKey][key] = val
  }
  Object.defineProperty(target, key, sharedPropertyDefinition)
}

他通过 sharedPropertyDefinition这个对象，定义了get和set函数，target就是vm，sourceKey是 _data，key就是我们在 data 中定义的字段，

我们通过 Object.defineProperty(target, key, sharedPropertyDefinition)执行 get set 方法， 其实就是 this.message = return this[sourceKey][key] = this._data.message，这样就会访问到我们的 message，

我们再梳理一下，也就是当我们在调用 this.message的时候，它实际上就是调用了 this._data.message，因为他是通过 proxy 做了层代理，这个才是他真正访问到的原因，当然我们的 props 也是这么做的，

在继续向下执行的话，就是observe，这个方法就是将定义到data中的对象变成一个响应式对象，该方法与响应式原理息息相关，我们会在响应式原理章节仔细分析该方法的实现。

# 总结

我们来对当前学习的这一小节做一下总结，我们在main.js 中去初始化 vue 的时候，实际上我们就走到了源码中的 new Vue()，在这个 function 的下面他还会执行一堆 mixin，我们重点分析了initMixin， initMixin中我们在 vue 的原型上面挂载了_init 方法，然后在这个方法里面做了一系列的初始化工作，包括 uid，merge options，再往下就是初始化函数，我们又重点分析了 initState，因为这个 initState 与我们这一小节要分析的问题有关，在这个函数里面他分别拿到了我们的 methods，props，data，他们三个都去调用了不同的函数，然后我们分析了initData，

initData要求 data 应该是一个函数，然后这个函数他应该返回一个对象，在这一步他还调用了getData() 方法，

完成了对 data 的处理以后，我们再回到initData 中，接着就是做了一层循环，要求 methods，props，data 的命名不能重复，因为最终我们要把这三个挂载到 vue 的原型上面去，

然后在这里我们调用了一个函数 proxy，这个方法中调用了 Object.defineProperty的 get set 方法，实现了我们所说的 this.message === this._data.message。

在 _init 方法的最后我们还实现了 observe响应式，后面的章节我们会继续分析。
